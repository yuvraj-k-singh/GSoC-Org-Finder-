Contributor Guide

Start Contributing with Confidence

New to open source or this project? Follow this quick path: learn the basics, set up locally, find a good first issue, and open your first PR.

1) Open Source Basics

Open source projects publish source code publicly so anyone can inspect, improve, and share it. Contribution usually starts with reading contribution guidelines and making small, focused changes.

  • Understand how licensing and collaboration work.
  • Start with documentation or beginner-friendly issues.
  • Communicate early through issues and PR comments.

2) Contribute to a GSoC Organization

  1. Pick 2–3 organizations that match your skills and interests.
  2. Read their ideas page, contribution guide, and communication channels.
  3. Set up the project locally and start with a small issue or documentation task.
  4. Open a focused pull request and respond to mentor feedback quickly.
  5. Document your contributions and turn them into a stronger GSoC proposal.

Tip: quality communication and small, consistent contributions usually matter more than a single large PR.

4) FAQ

Do I need prior GSoC experience to contribute?

No. Start with small docs or UI fixes, then move to larger tasks as you learn the codebase.

What should my first pull request look like?

Keep it focused on one change, reference the issue number, and include a short explanation of what you tested.

How do I find tasks I can start quickly?

Look for issues labeled good first issue and comment before starting work.
